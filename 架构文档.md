## 1. 系统概述

智能问答系统（Intelligent Q&A System）是一种能够理解用户自然语言提问，并从知识库中检索或生成准确答案的 AI 系统。本项目涵盖从**数据预处理 → 模型构建 → 训练 → 推理**的完整流程。

### 核心能力

| 能力 | 说明 |
|------|------|
| **问题理解** | 对用户输入进行意图分类、关键词提取 |
| **知识检索** | 从结构化/非结构化数据中检索相关信息 |
| **答案生成** | 基于检索结果生成或抽取精准答案 |
| **端到端流水线** | 覆盖数据到部署的全链路 |

## 3. 模块详解

### 3.1 语料预处理

将原始语料转换为模型可用的格式。

#### 处理流程

```
原始语料 → 读取语料 → 分词 → 构建词典 → 拆分问答对 → 保存
```

#### 子任务

| 任务 | 说明 |
|------|------|
| **读取语料** | 支持多种格式（JSON、CSV、TXT），加载原始问答数据 |
| **分词并构建词典** | 对中文文本进行分词（如 jieba），构建词表（vocab）和索引映射 |
| **拆分问题、回答和保存文件** | 将问答对分离，分别保存为训练所需的格式 |

#### 输入/输出

- **输入**：原始语料文件（`.json` / `.csv` / `.txt`）
- **输出**：分词后的序列文件、词典文件（`vocab.json`）

#### 补充细节

- 中文分词推荐使用 **jieba** 或 **HanLP**，也可使用基于 BPE 的 tokenizer
- 词典应包含特殊标记：`[PAD]`, `[UNK]`, `[CLS]`, `[SEP]`, `[MASK]`
- 建议设置最小词频阈值过滤低频词，控制词典大小在 30K–50K

### 3.2 问题理解

分析用户输入的自然语言问题，提取关键信息。

```
用户问题 ──▶ 问题分类 ──▶ 关键词提取 ──▶ 结构化查询
```

#### 问题分类

将问题归类到预定义的类别中，便于后续检索策略选择。

| 分类维度 | 示例类别 |
|----------|---------|
| 知识类型 | 事实型、推理型、建议型、定义型 |
| 领域 | 科技、历史、医学、法律、通用 |
| 复杂度 | 简单（单跳）、复杂（多跳） |

**实现方式**：
- 基于规则的分类（关键词匹配 + 正则）
- 基于文本分类模型（如 BERT-based classifier）
- Few-shot 分类（大模型推理）

#### 关键词提取

从问题中提取核心实体和关键词，用于检索。

**实现方式**：
- **TF-IDF**：传统方法，基于词频统计
- **TextRank**：基于图的排序算法
- **基于模型**：使用序列标注（BERT-BiLSTM-CRF）提取实体

#### 输入/输出

- **输入**：用户原始问题文本
- **输出**：问题类别标签 + 关键词列表

---

### 3.3 知识检索

根据问题理解阶段的输出，从知识库中查找相关信息。

```
查询 ──▶ 结构化检索 ──┐
                      ├──▶ 融合排序 → 候选段落
查询 ──▶ 非结构化检索 ─┘
```

#### 结构化信息检索

针对数据库、知识图谱等结构化数据源。

| 数据源 | 检索方式 |
|--------|---------|
| 关系型数据库 | SQL 查询，将自然语言转 SQL（NL2SQL） |
| 知识图谱 | SPARQL 查询，基于实体和关系匹配 |
| 表格数据 | 列名匹配 + 条件过滤 |

#### 非结构化信息检索

针对文档、网页等非结构化文本。

**实现方式**：
- **稀疏检索**：BM25 / TF-IDF，关键词匹配
- **稠密检索（Dense Retrieval）**：使用双编码器（Dual-Encoder）将问题和文档编码为向量，计算语义相似度
  - 常用模型：Sentence-BERT、DPR、Contriever
- **混合检索**：稀疏 + 稠密融合，兼顾精确匹配与语义理解

**补充——向量数据库**：
- 使用 FAISS、Milvus 或 Chroma 存储文档向量
- 支持近似最近邻（ANN）搜索，大幅提升检索速度
- 配合 RAG（检索增强生成）模式使用

#### 输入/输出

- **输入**：问题类别 + 关键词/向量
- **输出**：相关文档段落列表（按相关性排序）

---

### 3.4 答案生成

基于检索结果生成或抽取最终答案。

```
候选文档 ──▶ 答案抽取 ──▶ 答案生成 ──▶ 答案验证 ──▶ 最终答案
```

#### 答案抽取

从检索到的文档中定位答案片段。

**方式**：
- 基于阅读理解模型（BERT-based reading comprehension）
- 预测答案在文档中的起始和结束位置（Span Prediction）
- 适用于事实型问题

#### 答案生成

基于检索到的信息生成新的答案文本。

**方式**：
- **基于 seq2seq 模型**：T5、BART、LLaMA、ChatGLM 等
- **RAG（检索增强生成）**：检索 + 生成联合建模
- **提示工程 + 大模型**：将检索结果组装为 prompt，输入大模型生成答案

#### 答案验证

对生成的答案进行正确性校验。

**策略**：
- **自洽性检查**：多次采样生成，选择最一致的答案
- **事实核对（Fact-checking）**：验证答案中的每个声明是否在原文中有支撑
- **规则过滤**：排除长度异常、包含敏感词的答案
- **置信度评分**：给每个答案附带置信度分数，低分答案降级或丢弃

#### 输入/输出

- **输入**：候选文档段落 + 原始问题
- **输出**：最终答案文本 + 置信度分数

### 3.5 数据准备

将预处理后的数据组装为模型训练所需的批次。

```
词典文件 ──┐
           ├──▶ 创建 DataLoader → 批次数据
训练数据 ──┘
```

| 步骤 | 说明 |
|------|------|
| 加载词典和数据 | 读取 vocab.json 和分词后的序列数据 |
| 构建 Dataset | 封装为 PyTorch Dataset，实现 `__getitem__` |
| 创建 DataLoader | 配置 batch_size、shuffle、num_workers，支持动态 padding |

#### 补充细节

- **padding 策略**：同批次内按最大长度 padding，减少显存浪费
- **mask 机制**：生成 attention_mask，区分有效 Token 与 padding Token
- **标签构建**：根据任务类型构建标签（分类标签、Span 位置、生成序列等）

### 3.6 参数设置

集中管理系统运行所需的超参数和配置。

| 参数类别 | 示例 |
|----------|------|
| 数据参数 | 最大序列长度、词典路径、数据文件路径 |
| 模型参数 | 隐藏层维度、注意力头数、层数、dropout |
| 训练参数 | learning rate、batch size、epochs、warmup steps |
| 优化器参数 | AdamW betas、weight decay、学习率调度策略 |
| 检索参数 | top-k 数量、检索阈值、向量维度 |

**实现建议**：
- 使用 `argparse` 或 `yaml` / `json` 配置文件管理
- 使用 `dataclass` 定义参数类，支持类型检查和自动补全
- 示例：`configs/default.yaml`

### 3.7 模型构建

定义问答系统的神经网络架构。

```
输入层 → Embedding层 → Encoder层 → 输出层
```

#### 核心模型

| 组件 | 说明 |
|------|------|
| **嵌入层（Embedding）** | Token Embedding + Position Embedding + Segment Embedding |
| **编码器（Encoder）** | Transformer / BERT 堆叠层 |
| **输出层** | 根据任务不同（分类头、生成头、Span 预测头） |

#### 构建选择

| 方案 | 适用场景 | 说明 |
|------|---------|------|
| **从零构建** | 教学/学习目的 | 自行实现 Transformer 编码器，理解底层原理 |
| **微调预训练模型** | 实际项目 | 基于 BERT / RoBERTa / ChatGLM / Qwen |
| **LoRA 微调** | 资源受限 | 冻结原模型参数，插入低秩适配矩阵 |

> OCR 识别中的"构建58925e9模型"应为**构建 BERT 模型**或类似预训练模型。

#### 优化器和损失函数

| 组件 | 推荐选择 |
|------|---------|
| **优化器** | AdamW（Adam with Weight Decay） |
| **学习率调度** | linear warmup + cosine decay |
| **损失函数** | 分类 → CrossEntropyLoss；生成 → CrossEntropyLoss (label smoothing)；检索 → Contrastive Loss (InfoNCE) |

#### LoRA（Low-Rank Adaptation）

- 冻结预训练模型权重，在注意力层注入低秩矩阵
- 参数量仅为原模型 0.1%–1%，大幅降低显存需求
- 适用于单卡训练大模型（7B+）

### 3.8 模型训练

执行模型的训练循环。

```
设置训练步数 → 前向传播 → 计算损失 → 反向传播 → 优化器更新 → 保存检查点
```

| 步骤 | 说明 |
|------|------|
| **设置训练步** | 配置总训练步数、warmup 步数、日志/保存间隔 |
| **前向传播** | 输入批次数据 → 模型推理 → 计算损失 |
| **反向传播** | `loss.backward()` → 梯度裁剪（grad clip） |
| **优化器更新** | `optimizer.step()` → `scheduler.step()` → 零梯度 |
| **训练并保存模型** | 每个 checkpoint 保存权重、优化器状态、配置 |

#### 补充细节

- **混合精度训练**：使用 `torch.cuda.amp` 或 DeepSpeed，加速训练
- **梯度累积**：模拟更大的 batch size，适用于显存受限场景
- **Early Stopping**：监控验证集 loss，连续 N 轮不下降则提前停止
- **检查点策略**：保存最佳模型（best.pt）和最新模型（last.pt）

### 3.9 模型测试

在测试集上评估训练好的模型。

| 测试项 | 说明 |
|--------|------|
| 准确率 | 预测答案与标准答案的匹配度 |
| 召回率 | 正确答案被成功检索到的比例 |
| 推理速度 | 单条查询的响应时间（ms/query） |
| 资源占用 | GPU 显存 / CPU 内存消耗 |

**测试流程**：
```
加载训练好的模型 → 加载测试数据 → 逐条推理 → 计算指标 → 输出测试报告
```

### 3.10 模型评价

对模型进行综合评估，生成详细评价报告。

#### 评价指标

| 指标 | 说明 |
|------|------|
| **Exact Match (EM)** | 预测答案与标准答案完全一致的比例 |
| **F1 Score** | 预测答案与标准答案的 Token 级别重叠度 |
| **Precision@K** | 检索结果中前 K 条的相关性 |
| **MRR (Mean Reciprocal Rank)** | 第一个正确答案在检索结果中的平均排名倒数 |
| **BLEU / ROUGE** | 生成式答案的文本相似度指标 |
| **Perplexity** | 生成模型的困惑度 |
| **人工评价** | 流畅性、有用性、安全性 |

#### 分析工具

- **混淆矩阵**：分析各分类的错误模式
- **PR 曲线**：不同阈值下的精确率-召回率曲线
- **错误案例收集**：归纳模型常见错误类型，指导后续优化

## 4. 技术选型建议

### 开发环境

| 组件 | 推荐 |
|------|------|
| 编程语言 | Python 3.10+ |
| 深度学习框架 | PyTorch 2.x |
| 分词工具 | jieba / transformers tokenizer |
| 向量检索 | FAISS (Facebook AI) |
| 实验跟踪 | TensorBoard / WandB |

### 模型选择

| 项目类型 | 推荐模型 | 说明 |
|---------|---------|------|
| 分类/阅读理解 | BERT-base-Chinese | 12层，1.1亿参数，适合大多数任务 |
| 检索 | Sentence-BERT / bge-base-zh | 生成语义向量 |
| 生成式问答 | Qwen2 / ChatGLM3 / LLaMA-3-Chinese | 适合开放域生成 |
| 资源受限 | BERT-tiny + LoRA | 显存需求 < 4GB |

### 数据集

| 数据集 | 用途 |
|--------|------|
| DuReader | 中文阅读理解 |
| WebQA | 中文社区问答 |
| CMRC 2018 | 中文机器阅读理解 |
| 自建数据集 | 根据项目领域自行构建 |

## 6. 运行流程

### 训练阶段

```bash
# 1. 语料预处理
python modules/preprocess.py --input data/raw/corpus.json --output data/processed/

# 2. 训练模型
python train.py --config configs/train.yaml
```

### 推理阶段

```bash
# 启动问答服务
python predict.py --question "中国的首都是哪里？" --model checkpoints/best.pt

# 输出: 北京
```

### 端到端交互流程

```
用户输入: "什么是机器学习？"
                                              │
  ┌── 问题理解 ──────────────────────────────▼──────────────────────┐
  │  分类: [知识定义型]          关键词: ["机器学习", "定义"]        │
  └─────────────────────────────────────────────────────────────────┘
                                              │
  ┌── 知识检索 ──────────────────────────────▼──────────────────────┐
  │  结构化检索: [ML Wiki 实体匹配]                                 │
  │  非结构化检索: [BM25 命中2篇，Dense 检索命中3篇]                │
  │  融合排序: [Top-3 候选段落]                                    │
  └─────────────────────────────────────────────────────────────────┘
                                              │
  ┌── 答案生成 ──────────────────────────────▼──────────────────────┐
  │  抽取: [定位到"机器学习是人工智能的一个分支..."]                 │
  │  生成: [整合背景信息，生成完整定义]                             │
  │  验证: [事实核对通过 ✓  置信度: 0.95]                          │
  └─────────────────────────────────────────────────────────────────┘
                                              │
                      输出: "机器学习是人工智能的一个分支，
                      它使计算机系统能够从数据中学习和改进..."
```
