## 1. 系统概述

本项目设计的是一个**面向中文场景的智能问答系统**，用于接收用户自然语言问题，并结合知识库中的内容返回较为准确、简洁、可解释的答案。

系统整体采用“**问题理解 → 知识检索 → 答案生成 → 结果验证**”的处理链路。  
其中，前半部分负责找对信息，后半部分负责把信息组织成可读答案。

### 设计目标

- 支持中文问题输入
- 能够对问题进行分类和关键词提取
- 支持结构化数据与非结构化文本的联合检索
- 支持抽取式问答与生成式问答两种模式
- 具备可训练、可测试、可扩展的工程结构

### 系统特点

| 特点 | 说明 |
|------|------|
| 模块化 | 各功能模块解耦，便于维护与替换 |
| 可扩展 | 可替换检索器、生成模型和评价指标 |
| 可落地 | 兼顾训练、推理、测试和部署流程 |
| 可解释 | 检索结果与答案来源可追踪 |

### 适用场景

- 校园/课程项目中的知识问答系统
- 企业内部知识库问答
- FAQ 问答
- 基于文档的检索增强问答（RAG）

## 2. 总体架构

系统分为**在线推理流程**和**离线训练流程**两部分。

### 在线推理流程

```text
用户输入
   │
   ▼
问题理解
   ├─ 问题分类
   └─ 关键词提取
   │
   ▼
知识检索
   ├─ 结构化检索
   └─ 非结构化检索
   │
   ▼
答案生成
   ├─ 答案抽取
   ├─ 答案生成
   └─ 答案验证
   │
   ▼
最终答案输出
```

### 离线训练流程

```text
原始语料
   │
   ▼
语料预处理
   │
   ▼
数据准备
   │
   ▼
模型构建
   │
   ▼
模型训练
   │
   ▼
模型测试与评价
```

### 架构说明

- **在线流程**关注实时响应，重点是准确性和速度。
- **离线流程**关注模型效果提升，重点是数据质量和训练稳定性。
- 两条流程共享同一套配置、模型与数据标准，保证训练和推理的一致性。

## 3. 核心模块设计

### 3.1 语料预处理

语料预处理的目标是把原始数据转换为模型可直接使用的格式。

#### 处理流程

```text
原始语料 → 读取数据 → 清洗文本 → 分词/编码 → 构建词表 → 保存处理结果
```

#### 主要任务

| 任务 | 说明 |
|------|------|
| 读取语料 | 支持 JSON、CSV、TXT 等格式 |
| 文本清洗 | 去除空白字符、重复样本、异常符号 |
| 分词或编码 | 中文分词或使用 tokenizer 直接编码 |
| 构建词表 | 生成词典及 token-id 映射 |
| 数据保存 | 输出预处理后的训练数据与词表文件 |

#### 输入与输出

- **输入**：原始问答语料
- **输出**：处理后的序列数据、词表文件、统计信息

#### 说明

- 中文分词可使用 `jieba`、`HanLP` 或 `transformers tokenizer`
- 建议保留特殊标记：`[PAD]`、`[UNK]`、`[CLS]`、`[SEP]`
- 对低频词进行过滤，避免词表过大
- 若采用预训练模型，可直接使用其自带分词器，减少额外词表设计成本

---

### 3.2 问题理解

问题理解模块用于识别用户问题的类型，并提取后续检索所需的关键信息。

#### 处理流程

```text
用户问题 → 问题分类 → 关键词/实体提取 → 结构化查询表示
```

#### 问题分类

问题分类的作用是帮助系统决定采用哪种检索和回答策略。

| 分类维度 | 示例 |
|----------|------|
| 知识类型 | 定义型、事实型、推理型、建议型 |
| 领域 | 通用、科技、历史、医学、法律等 |
| 复杂度 | 单跳问题、多跳问题 |

#### 关键词/实体提取

| 方法 | 说明 |
|------|------|
| 基于规则 | 通过关键词、正则和词典匹配提取 |
| 传统方法 | TF-IDF、TextRank |
| 模型方法 | 序列标注、BERT 类实体识别模型 |

#### 输入与输出

- **输入**：用户原始问题
- **输出**：问题类别、关键词列表、实体信息

#### 作用

- 决定检索策略
- 缩小检索范围
- 提高后续答案生成的准确率

---

### 3.3 知识检索

知识检索模块负责从知识库中找到与问题最相关的内容。

#### 检索流程

```text
问题表示 → 候选召回 → 相关性排序 → 输出候选段落
```

#### 结构化检索

适用于数据库、知识图谱、表格等结构化数据。

| 数据源 | 检索方式 |
|--------|---------|
| 关系型数据库 | SQL 查询或 NL2SQL |
| 知识图谱 | 实体匹配 + 关系查询 |
| 表格数据 | 列名映射 + 条件过滤 |

#### 非结构化检索

适用于文档、网页、FAQ、论文等文本内容。

| 方法 | 说明 |
|------|------|
| 稀疏检索 | BM25、TF-IDF，适合关键词匹配 |
| 稠密检索 | 双编码器向量检索，适合语义匹配 |
| 混合检索 | 稀疏检索 + 稠密检索融合 |
| 向量库 | FAISS、Milvus、Chroma 等 |

#### 推荐方案

在课程项目中，建议采用以下简化方案：

- **第一阶段**：BM25 做初筛
- **第二阶段**：向量召回做语义补充
- **第三阶段**：基于相似度重排序

#### 输入与输出

- **输入**：问题类别、关键词、向量表示
- **输出**：排序后的候选文档段落

---

### 3.4 答案生成

答案生成模块根据检索结果组织最终回答。

#### 处理流程

```text
候选段落 → 答案抽取/生成 → 答案验证 → 最终答案
```

#### 答案抽取

适用于事实型、定位明确的问题。

- 基于阅读理解模型抽取答案片段
- 预测答案起始位置和结束位置
- 输出精确的文本片段

#### 答案生成

适用于需要整合多段信息或自然表达的问题。

- 使用 seq2seq 模型进行文本生成
- 使用 RAG 方式将检索结果作为上下文输入模型
- 使用提示词模板组织输入内容

#### 答案验证

答案验证用于降低模型“胡编”或答非所问的风险。

| 方法 | 说明 |
|------|------|
| 事实核对 | 检查答案是否能被检索结果支撑 |
| 一致性检查 | 多次生成结果是否一致 |
| 规则过滤 | 过滤过长、过短或异常输出 |
| 置信度评分 | 为答案附加可信度分值 |

#### 输入与输出

- **输入**：候选文档段落、用户问题
- **输出**：最终答案、置信度评分、来源信息

---

### 3.5 数据准备

数据准备模块用于把预处理后的语料组装成训练所需的数据形式。

#### 主要步骤

| 步骤 | 说明 |
|------|------|
| 加载数据 | 读取处理后的样本和词表 |
| 构建数据集 | 封装为 Dataset，支持索引访问 |
| 构建 DataLoader | 设置 batch size、shuffle、num_workers |
| 动态 padding | 按 batch 内最长序列补齐 |
| 构建标签 | 根据任务类型生成分类标签、Span 标签或生成标签 |

#### 关键点

- 对变长序列进行统一处理
- 生成 attention mask，区分有效 token 与 padding
- 保证训练、验证、测试集格式一致
- 如需提升泛化能力，可加入数据增强或负样本构造

---

### 3.6 参数配置

参数配置模块用于统一管理模型训练和推理所需的配置项。

#### 参数分类

| 类别 | 示例 |
|------|------|
| 数据参数 | 数据路径、最大长度、词表路径 |
| 模型参数 | hidden size、层数、注意力头数 |
| 训练参数 | learning rate、batch size、epochs |
| 检索参数 | top-k、召回阈值、向量维度 |
| 推理参数 | 最大输出长度、温度系数、beam size |

#### 管理方式

- 使用 `yaml` 或 `json` 管理配置文件
- 使用 `argparse` 支持命令行覆盖参数
- 使用 `dataclass` 统一定义配置结构

#### 建议

将常用配置拆分为：
- `configs/default.yaml`
- `configs/train.yaml`
- `configs/infer.yaml`

这样便于区分训练与推理阶段的不同参数。

### 3.7 模型构建

模型构建模块定义系统的核心神经网络结构。

#### 基本结构

```text
输入层 → Embedding 层 → Encoder 层 → 任务输出头
```

#### 主要组件

| 组件 | 说明 |
|------|------|
| Embedding 层 | 表示 token、位置和语义信息 |
| Encoder 层 | 提取上下文特征 |
| 输出头 | 根据任务类型输出分类、抽取或生成结果 |

#### 模型方案

| 方案 | 适用场景 | 说明 |
|------|----------|------|
| 从零构建 | 教学、原理验证 | 自己实现基础结构 |
| 微调预训练模型 | 课程项目、实际开发 | 直接使用 BERT 类模型 |
| LoRA 微调 | 显存受限场景 | 只训练少量适配参数 |

#### 损失函数与优化器

| 组件 | 推荐选择 |
|------|----------|
| 优化器 | AdamW |
| 学习率调度 | warmup + 线性衰减或余弦衰减 |
| 分类损失 | CrossEntropyLoss |
| 抽取损失 | 起止位置交叉熵 |
| 生成损失 | 序列交叉熵 |
| 检索损失 | 对比学习损失（如 InfoNCE） |

#### 说明

如果项目目标是完成课程设计，推荐采用“**预训练模型 + 轻量微调**”的方案，工作量和效果都比较平衡。

### 3.8 模型训练

模型训练模块负责执行参数更新和模型收敛过程。

#### 训练流程

```text
读取批次数据 → 前向传播 → 计算损失 → 反向传播 → 参数更新 → 保存模型
```

#### 训练步骤

| 步骤 | 说明 |
|------|------|
| 前向传播 | 将输入送入模型，得到预测结果 |
| 计算损失 | 与真实标签计算误差 |
| 反向传播 | 计算梯度并回传 |
| 参数更新 | 使用优化器更新参数 |
| 保存检查点 | 保存最佳模型与最新模型 |

#### 可选优化

- 混合精度训练
- 梯度裁剪
- 梯度累积
- Early Stopping
- 模型断点续训

#### 训练输出

- 模型权重文件
- 训练日志
- 损失曲线和指标曲线
- 最优 checkpoint

---

### 3.9 模型测试

模型测试模块用于验证模型在测试集上的实际效果。

#### 测试目标

- 验证功能是否正常
- 检查推理结果是否稳定
- 评估模型在未知数据上的表现

#### 测试内容

| 测试项 | 说明 |
|------|------|
| 准确率 | 预测结果与标准答案一致的比例 |
| 召回率 | 正确答案被找回的比例 |
| 推理速度 | 单条问题平均响应时间 |
| 资源占用 | 显存和内存占用情况 |

#### 测试流程

```text
加载模型 → 读取测试集 → 批量推理 → 统计指标 → 输出测试报告
```

### 3.10 模型评价

模型评价模块用于对系统效果进行综合分析。

#### 评价指标

#### 检索任务指标

| 指标 | 说明 |
|------|------|
| Precision@K | 前 K 个结果中相关结果的比例 |
| Recall@K | 前 K 个结果覆盖正确答案的能力 |
| MRR | 第一个正确结果的平均排名倒数 |

#### 抽取式问答指标

| 指标 | 说明 |
|------|------|
| Exact Match (EM) | 预测答案与标准答案完全一致的比例 |
| F1 Score | 预测答案与标准答案的重合度 |

#### 生成式问答指标

| 指标 | 说明 |
|------|------|
| BLEU | 生成结果与参考答案的字面相似度 |
| ROUGE | 对生成文本召回程度的衡量 |
| 人工评价 | 流畅性、准确性、可读性 |

#### 分析方法

- 混淆矩阵
- 错误案例分析
- 检索失败案例分析
- 生成答案偏差分析

## 4. 接口与部署设计

为了让系统更完整，建议补充接口与部署层设计。

### 接口设计

| 接口 | 功能 |
|------|------|
| `/predict` | 输入问题，返回答案 |
| `/search` | 返回候选检索结果 |
| `/health` | 检查服务状态 |
| `/reload` | 热更新模型或索引 |

### 部署方式

- 命令行部署：适合开发和调试
- Web 服务部署：适合演示和使用
- 模型服务化：适合后续扩展为完整应用

### 异常处理

- 问题为空时返回提示信息
- 检索不到结果时启用兜底回答
- 模型加载失败时给出明确报错
- 对超长输入进行截断或分段处理

## 5. 技术选型建议

### 开发环境

| 组件 | 推荐 |
|------|------|
| 编程语言 | Python 3.10+ |
| 深度学习框架 | PyTorch 2.x |
| 分词工具 | jieba / transformers tokenizer |
| 向量检索 | FAISS |
| 实验记录 | TensorBoard / WandB |

### 推荐模型

| 场景 | 推荐方案 | 说明 |
|------|----------|------|
| 分类与抽取 | BERT-base-Chinese | 适合分类、匹配、抽取任务 |
| 检索 | Sentence-BERT / bge-base-zh | 适合语义召回 |
| 生成式问答 | Qwen / ChatGLM 类模型 | 适合开放式回答 |
| 资源受限 | 小模型 + LoRA | 降低显存与训练成本 |

### 数据集

| 数据集 | 用途 |
|--------|------|
| DuReader | 中文阅读理解 |
| CMRC 2018 | 中文机器阅读理解 |
| WebQA | 社区问答 |
| 自建数据集 | 适配课程项目主题 |

## 7. 运行流程

### 训练阶段

```bash
# 1. 语料预处理
python modules/preprocess.py --input data/raw/corpus.json --output data/processed/

# 2. 训练模型
python train.py --config configs/train.yaml
```

### 推理阶段

```bash
python predict.py --question "中国的首都是哪里？" --model checkpoints/best.pt
```

### 端到端流程示例

```text
用户输入：“什么是机器学习？”
   │
   ▼
问题理解：
- 分类：定义型问题
- 关键词：机器学习、定义
   │
   ▼
知识检索：
- BM25 命中相关文档
- 向量检索补充语义匹配
- 输出 Top-K 候选段落
   │
   ▼
答案生成：
- 抽取或生成答案
- 进行事实核对
- 输出置信度
   │
   ▼
最终输出：
“机器学习是人工智能的一个分支，它使计算机能够从数据中学习规律并改进性能。”
```
