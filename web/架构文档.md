## 1. 系统概述

### 定位

支持用户注册登录、管理个人模型（检索式/生成式）、在线问答评测、多人对比打分、排行榜展示的全功能 Web 平台。

### 核心链路

```
用户注册 → 上传模型 → 问答测试 → 多模型对比 → 打分 → 排行榜
                               ↑
                         公开模型广场
```

### 两种模型类型

| 维度 | 检索式 (retrieval) | 生成式 (generative) |
|------|-------------------|-------------------|
| 原理 | TF-IDF 匹配预设问答对 | 千问模型 LoRA 微调生成 |
| 输入文件 | JSON 问答对（`question`+`answer`） | LoRA 权重 zip（`adapter_config.json`） |
| 回答方式 | 返回数据集中最匹配的预设答案 | 模型自行组织语言生成答案 |
| 上下文 | ❌ 无记忆，一问一答 | ✅ 支持多轮上下文对话 |
| 速度 | CPU，毫秒级 | GPU，秒级 |
| 灵活度 | 只能回答数据集内的问题 | 可泛化到相似问题 |
| 典型场景 | 客服 FAQ、固定知识库 | 智能聊天、开放式问答 |

> **一句话区别**：检索式是查字典，生成式是真聊天。

## 2. 数据模型

```sql
-- 用户
CREATE TABLE users (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    username      TEXT    UNIQUE NOT NULL,
    password_hash TEXT    NOT NULL,
    created_at    TEXT    DEFAULT (datetime('now'))
);

-- 模型
CREATE TABLE models (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id     INTEGER NOT NULL REFERENCES users(id),
    name        TEXT    NOT NULL,
    type        TEXT    NOT NULL CHECK(type IN ('retrieval', 'generative')),
    file_path   TEXT,                    -- 上传文件的存储路径
    description TEXT    DEFAULT '',
    public      INTEGER DEFAULT 0,       -- 0=私有的, 1=公开
    created_at  TEXT    DEFAULT (datetime('now')),
    updated_at  TEXT    DEFAULT (datetime('now')),
    downloads   INTEGER DEFAULT 0,       -- 被其他用户调用次数
    avg_score   REAL    DEFAULT 0.0      -- 用户评分均值（缓存）
);

-- 对比打分
CREATE TABLE ratings (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id       INTEGER NOT NULL REFERENCES users(id),       -- 打分者
    model_id      INTEGER NOT NULL REFERENCES models(id),
    question      TEXT    NOT NULL,
    answer        TEXT    NOT NULL,
    score         INTEGER NOT NULL CHECK(score BETWEEN 1 AND 5),
    session_id    TEXT,                    -- 同一次对比会话的标识
    created_at    TEXT    DEFAULT (datetime('now'))
);

-- 使用统计
CREATE TABLE usage_logs (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    model_id    INTEGER NOT NULL REFERENCES models(id),
    caller_id   INTEGER REFERENCES users(id),
    api_path    TEXT,
    latency_ms  INTEGER,
    created_at  TEXT    DEFAULT (datetime('now'))
);

-- 官方评测分数（管理员配置）
CREATE TABLE official_scores (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    model_id     INTEGER NOT NULL REFERENCES models(id),
    test_set     TEXT    NOT NULL,
    metric       TEXT    NOT NULL,
    score        REAL    NOT NULL,
    created_at   TEXT    DEFAULT (datetime('now'))
);
```

### 索引

```sql
CREATE INDEX idx_models_user ON models(user_id);
CREATE INDEX idx_models_type ON models(type);
CREATE INDEX idx_ratings_model ON ratings(model_id);
CREATE INDEX idx_usage_model ON usage_logs(model_id);
CREATE INDEX idx_official_model ON official_scores(model_id);
```

## 3. API 设计

### 认证

| 方法 | 端点 | 说明 |
|------|------|------|
| POST | `/api/auth/register` | 注册：`{ username, password }` |
| POST | `/api/auth/login` | 登录：`{ username, password }` → JWT |

### 模型管理（需登录）

| 方法 | 端点 | 说明 |
|------|------|------|
| GET | `/api/models` | 我的模型列表 |
| POST | `/api/models` | 创建模型：`{ name, type, description, public }` |
| POST | `/api/models/{id}/upload` | 上传文件（LoRA zip / JSON 数据） |
| GET | `/api/models/{id}` | 模型详情 |
| DELETE | `/api/models/{id}` | 删除模型 |
| GET | `/api/models/public` | 全体公开模型列表 |

### 问答

| 方法 | 端点 | 说明 |
|------|------|------|
| POST | `/api/qa/ask` | 单模型问答：`{ model_id, question }` → `{ answer, confidence, latency }` |
| POST | `/api/qa/compare` | 多模型对比：`{ model_ids: [...], question }` → `[{ model_id, name, answer, confidence }, ...]` |

### 评分

| 方法 | 端点 | 说明 |
|------|------|------|
| POST | `/api/ratings` | 提交评分：`{ model_id, question, answer, score, session_id }` |

### 排行榜

| 方法 | 端点 | 说明 |
|------|------|------|
| GET | `/api/ranking` | 排行榜：`?type=all\|retrieval\|generative&sort=avg_score\|downloads\|official&period=7d\|30d\|all` |

### 管理（预留）

| 方法 | 端点 | 说明 |
|------|------|------|
| GET | `/api/admin/pending-models` | 待审核模型列表 |
| POST | `/api/admin/official-score` | 录入官方评测分数 |

## 5. 问答流程

### 检索式模型流程

```
用户选择模型 + 输入问题
  → 定位模型关联的 JSON 数据文件
  → 构建（或加载缓存）TF-IDF 索引
  → 计算相似度 → Top-K 候选
  → 返回答案 + 置信度
```

> 注：索引在模型上传时预构建，推理时秒级响应。

### 生成式模型流程

```
用户选择模型 + 输入问题
  → 加载基座模型（Qwen2.5-3B-Instruct，全局一份）
  → 如果 GPU 上已有其他 LoRA，先卸载
  → 加载用户上传的 LoRA 权重（PeftModel.from_pretrained）
  → 推理生成答案
  → 返回结果（同时后台记录延迟）
  → 释放 LoRA 权重
```

> 加载耗时约 1-2 秒，推理耗时取决于问题长度。

## 6. 加载策略

### 方案A：即用即加载（选用）

```
请求到来
  ├─ 检索式 → 索引已在内存 / SQLite，直接查
  └─ 生成式 → 当前 GPU LoRA 是目标？
                  ├─ 是 → 直接推理
                  └─ 否 → 卸载当前 LoRA
                          → PeftModel.from_pretrained 加载目标 LoRA
                          → 推理 → 返回
                          → 标记"可卸载"（下个请求覆盖）
```

**优势：** 显存省，只占一份 LoRA 空间
**代价：** 每次切换模型需 1-2s 加载时间

### 基座模型策略

- Qwen2.5-3B-Instruct (~6GB) 常驻 GPU
- LoRA 权重 (~50MB) 按需加载
- 如果 GPU 显存不足：回退 CPU 推理（慢但可用）

## 7. 排行榜机制

### 子类

| Tab | 排序依据 | 说明 |
|-----|---------|------|
| 综合 | avg_score | 用户评分的均值（置信度加权） |
| 热度 | downloads + api_calls | 调用量越高越靠前 |
| 官方 | official_score | 管理员在测试集上跑的客观分数 |
| 检索式 | filter type=retrieval | 只看检索式模型 |
| 生成式 | filter type=generative | 只看生成式模型 |

### 评分聚合

```python
avg_score = sum(r.score * weight) / sum(weight)
# weight = 1 (普通用户) 或 3 (管理员)
```

### 排行榜缓存

- 每小时快照一次，避免每次都从头聚合
- 每次新评分触发增量更新

## 9. 技术选型

| 层 | 技术 | 说明 |
|----|------|------|
| Web 框架 | FastAPI | 异步、自动文档 |
| 数据库 | SQLite | 零配置，文件级 |
| 认证 | PyJWT | 无状态 Token |
| 前端 | 纯 HTML + CSS + JS (SPA) | 无框架依赖，轻量 |
| 生成式模型 | Transformers + PEFT + PyTorch | Qwen2.5 LoRA 推理 |
| 检索式引擎 | scikit-learn / jieba | TF-IDF |
